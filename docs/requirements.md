# 書籍閲覧・管理ツール 要件定義書

2026/4/26 作成 v1.0.0

## 1. はじめに

### 1.1 背景

電子書籍やPDF資料は端末ごとに管理が分散しやすく、閲覧環境や読書状況の管理が煩雑になりやすい。
本アプリケーションは、自身が正当に保有し、DRM等の技術的保護が施されていないEPUBファイルおよびPDFファイルを対象に、Webブラウザ上で閲覧・管理できる環境を提供することを目的とする。

### 1.2 目的

本アプリケーションの目的は、ユーザーが保有するEPUB/PDFファイルをアップロードし、ブラウザ上で閲覧・管理できる仕組みを提供することである。

主な目的は以下のとおりである。

- 書籍ファイルを一元管理できる
- ブラウザ上でEPUB/PDFを閲覧できる
- 読書状態や進捗を管理できる
- ユーザー権限に応じて閲覧可能な書籍を制御できる

### 1.3 範囲

本アプリケーションでは、以下を対象範囲とする。

- EPUBファイルのアップロード
- PDFファイルのアップロード
- 書籍情報の登録・編集・削除
- 書籍一覧の表示
- ブラウザ上での書籍閲覧
- 読書状態の管理
- ユーザー認証
- ユーザー権限による閲覧制御

一方で、以下は対象外とする。

- DRMで保護された電子書籍の解除・変換
- 電子書籍の販売機能
- 外部電子書籍ストアとの連携
- 全文検索機能
- OCR機能

### 1.4 用語定義

- **ユーザー**: 本アプリケーションを利用する個人。書籍のアップロードや閲覧を行う。
- **書籍**: EPUBまたはPDF形式のファイル。ユーザーがアップロードし、管理・閲覧する対象。
- **読書状態**: 書籍の閲覧状況を示す情報。例: 未読、読書中、読了。
- **ユーザー権限**: ユーザーがアクセスできる書籍や機能を制御するための設定。例: 管理者、一般ユーザー。

### 1.5 想定ユーザー

以下のニーズを持つユーザーを対象とする

- 個人で電子書籍を管理したいユーザー
- 複数の端末で同じ書籍を閲覧したいユーザー
- 読書進捗を管理したいユーザー
- 書籍の閲覧制御を行いたいユーザー

## 2. 機能要件

アプリケーションが使用できることを第一とし、
実装の重要度は以下で定義した上で、順番に実装を行う。

0 -> 必須機能
1 -> 重要な機能
2 -> 便利な機能
3 -> 将来的に追加を検討する機能

### 2.1 ユーザー機能

ユーザー機能として以下の機能を有する。

- ユーザーは、自身に閲覧権限が付与された書籍のみ閲覧できる
- 管理者がアップロードした書籍の閲覧(0)
  - EPUB/PDFのページ送り機能(0)
- 読書状態の管理(0)
  - ページ位置の保存(0)
  - 読書状態の更新(0)
  - 読書状態の表示(0)
  - しおり機能(1)
  - ブックマーク機能(1)
- カテゴリ/タグによる絞り込み機能(1)
- タイトル/著者による検索機能(1)
- 登録日/タイトル/著者による並び替え機能(2)
- 読書メモ作成(2)
- レビュー機能(2)
- オフライン閲覧機能(3)

### 2.2 管理者機能

管理者機能として以下の機能を有する。

- 管理者は全ての書籍を閲覧・管理できる
- ユーザー機能の全ての機能(0)
- ユーザー管理(0)
- 書籍管理(0)
- 書籍のアップロード/削除(0)
- 書籍情報の登録/編集/削除(0)
  - APIを用いた書籍情報の自動取得(1)
- ユーザー権限の設定(1)
- アクセスログの確認(2)
- 共有URLの管理(2)

### 2.3 ゲストユーザー機能

ゲストユーザー機能として以下の機能を有する。

- 共有されたURLの範囲内での書籍の閲覧(2)
  - 期限設定可能な共有設定(2)
  - ログイン不要での閲覧(2)

## 3. 非機能要件

### 3.1 パフォーマンス

- 書籍情報の読み込みは1秒以内に完了すること
- 書籍の閲覧はページ送りが1秒以内に反応すること
- ファイルアップロードは200MBまで対応すること

### 3.2 セキュリティ

- 管理者と一般ユーザーでの機能権限の切り分けがおこなわれていること
- アップロードされた書籍ファイルは適切にサーバー上で管理され、外部からの不正アクセスが防止するための設定を行うこと
- 管理者のみが書籍のアップロードや削除を行えるようにすること
- ログインしていないユーザーは、共有URLを除き書籍にアクセスできないこと
- 一般ユーザーは許可された書籍のみ閲覧できること
- アップロードされたファイルは公開ディレクトリに直接配置しないこと
- ファイル取得時は認証・権限チェックを行うこと

### 3.3 可用性

- 24時間アクセス可能（開発環境を除く）
- サーバーダウン時は復旧可能な構成とする（再起動で復旧）

## 4. システム要件

### 4.1 ハードウェア要件

以下のハードウェア要件を満たすこと。

- サーバー: 3コアCPU、2GB RAM、50GBストレージ以上
- クライアント: Webブラウザが動作するPCやタブレット
- WEBブラウザ: 最新のGoogle Chrome

### 4.2 ソフトウェア要件

以下のソフトウェア要件を満たすこと。

- サーバーOS: Linux(CentOS Stream 10)
- サーバーサイド: Node.js
- クライアントサイド: React
- データベース: PostgreSQL
- その他: Nginx

### 4.3 ネットワーク要件

- インターネット接続: 必要

## 5. 制約条件

- 書籍ファイルはDRMで保護されていないものに限る
- 書籍ファイルの最大サイズは200MBとする

## 6. データ要件

本アプリケーションでは、以下の情報を管理する。

- ユーザー情報
- 書籍情報
- 書籍ファイル情報
- 読書状態
- しおり情報
- タグ情報
- 共有URL情報
- アクセスログ

## 7. エラーハンドリング

- 不正なファイル形式はアップロード時にエラーとする
- ファイルサイズ制限を超えた場合はアップロードを拒否する
- 権限のないユーザーがアクセスした場合はエラーを返す
